# GLuaManager

Десктопный менеджер аддонов для Garry's Mod.

Язык: [🇷🇺 Русский](./README_RU.md) | [🇺🇸 English](./README.md)

GLuaManager помогает держать рабочую папку в порядке: сканирует установленные аддоны, сверяет их с удалёнными метаданными, показывает доступные обновления и устанавливает новые аддоны по URL метаданных.

## 🔥 Как это работает

Здесь два основных сценария.

### 🔗 Поток данных

Схема короткая:

- **source** это URL, который хранится в настройках
- каждый source указывает на JSON с метаданными аддона
- в метаданных лежат `info` и прямой `url` на архив
- именно архив скачивается и устанавливается

Источник может лежать где угодно, если по URL отдаётся валидный JSON.

### 📦 Установленные аддоны

- выберите папку с аддонами Garry's Mod
- приложение сканирует папки, где есть `.addon`
- у каждого установленного аддона есть локальные метаданные
- рядом сохраняется служебная привязка к source, откуда аддон был установлен
- при проверке обновлений приложение снова загружает эти метаданные и сравнивает версии
- если версии отличаются, аддон помечается как обновляемый
- при обновлении приложение скачивает архив из поля `url` и накатывает его поверх локальной папки

### 🌐 Удалённый каталог

- добавьте один или несколько source URL в настройках
- каждый source указывает на JSON с метаданными аддона
- приложение загружает эти метаданные и собирает удалённый каталог
- в карточке аддона показываются метаданные, ссылка на архив, зависимости и локальный README, если он есть
- при установке сначала строится план зависимостей, и только потом начинается загрузка

> [!NOTE]
> GLuaManager не является package manager. Аддоны Garry's Mod живут в одном файловом пространстве, поэтому приложение делает упор на видимость, предупреждения о зависимостях и управляемую установку.

## 🧩 Формат метаданных

`.addon` это обычный JSON-файл в корне аддона.

Пример:

```json
{
  "info": {
    "name": "Test addon",
    "description": "Пример аддона для документации",
    "author": "darkfated"
  },
  "version": "1.0.0",
  "url": "https://example.com/test-addon.zip",
  "dependencies": [
    "https://example.com/library-one.json"
  ],
  "preserve": ["lua/autorun/myaddon_config.lua", "data", "materials/custom/**"]
}
```

Поля:

- `info.name` - название аддона
- `info.description` - короткое описание
- `info.author` - автор
- `version` - версия установленного и удалённого аддона
- `url` - прямая ссылка на архив с аддоном
- `dependencies` - список URL метаданных зависимостей
- `preserve` - файлы и пути, которые нельзя перезаписывать при обновлении

`preserve` поддерживает:

- отдельный файл, например `lua/autorun/myaddon_config.lua`
- целую папку, например `data`
- glob-выражение, например `materials/custom/**`

Все пути относительны к папке аддона.

Приложение ожидает только `http` или `https` URL. `README.md` всегда берётся локально из папки аддона, если файл существует.

## 🗂️ Источники и примеры

Источники хранятся списком URL метаданных.

Пример:

```json
[
  "https://example.com/mantle.json",
  "https://example.com/newspawnmenu.json",
  "https://example.com/thirdperson.json"
]
```

Если нужен живой пример, смотри папку `exampleMeta/` в этом репозитории. Там лежат метаданные для `mantle`, `newspawnmenu` и `thirdperson`, а дефолтный source уже указывает на них.

Эти примеры показывают формат, который можно держать где угодно, если URL отдаёт валидный JSON.

## ⚙️ Установка и обновления

Логика одна и та же: приложение всегда опирается на source метаданных.

- install: сначала собирается план зависимостей, потом скачивается архив из `url`
- update: метаданные повторно загружаются по сохранённому source URL и сравнивается `version`
- match: аддон уже актуален
- mismatch: аддон можно обновить

После установки GLuaManager сохраняет source рядом с аддоном, чтобы проверка обновлений оставалась привязана к тому же источнику.

## 🛠️ Требования для сборки

На Debian / Ubuntu Linux нужны такие пакеты:

```bash
sudo apt update
sudo apt install libwebkit2gtk-4.1-dev \
  build-essential \
  curl \
  wget \
  file \
  libxdo-dev \
  libssl-dev \
  libayatana-appindicator3-dev \
  librsvg2-dev
```

Ещё нужен `tauri-cli`:

```bash
cargo install tauri-cli
```

Установите зависимости фронтенда:

```bash
cd ui
npm install
```

## ▶️ Запуск

Разработка:

```bash
cd src-tauri
cargo tauri dev
```

Сборка:

```bash
cd src-tauri
cargo tauri build
```
