# GLuaManager

Десктопный менеджер аддонов для Garry's Mod.

Язык: [🇷🇺 Русский](./README_RU.md) | [🇺🇸 English](./README.md)

GLuaManager помогает держать рабочую папку в порядке. Он сканирует установленные аддоны, сверяет их с удалёнными метаданными, показывает обновления и устанавливает аддоны по URL метаданных или через индексы источников.

## 🔥 Как это работает

Есть два пути: индексы источников в настройках и прямой URL метаданных на главной вкладке.

### 🔗 Поток данных

Схема простая:

- **source** это JSON-индекс в настройках
- каждый source содержит ссылки на JSON-метаданные аддонов
- в метаданных лежат `info` и прямой `url` на архив
- архив скачивается и устанавливается

Если известен один URL метаданных, используйте кнопку **Добавить аддон** на главной вкладке.

### 📦 Установленные аддоны

- выберите папку с аддонами Garry's Mod
- приложение сканирует папки, где есть `.addon`
- у каждого установленного аддона есть локальные метаданные
- рядом сохраняется URL метаданных
- при проверке обновлений приложение загружает этот URL и сравнивает версии
- если версии отличаются, аддон помечается как обновляемый
- при обновлении приложение скачивает архив из поля `url` и накатывает его поверх локальной папки

### 🌐 Удалённый каталог

- добавьте один или несколько URL индекса источника в настройках
- каждый индекс указывает на JSON-метаданные аддонов
- приложение загружает эти метаданные и собирает удалённый каталог
- если нужен один конкретный аддон, откройте его через **Добавить аддон** на главной вкладке
- в карточке показываются метаданные, ссылка на архив, зависимости и локальный README, если он есть
- установка сначала строит план зависимостей, потом запускает загрузку

> [!NOTE]
> GLuaManager не является package manager. Аддоны Garry's Mod живут в одном файловом пространстве, поэтому приложение делает упор на видимость, предупреждения о зависимостях и управляемую установку.

## 🧩 Формат метаданных

`.addon` это обычный JSON-файл в корне аддона.

Пример:

```json
{
  "info": {
    "name": "Test addon",
    "description": "Пример аддона для документации",
    "author": "darkfated"
  },
  "version": "1.0.0",
  "url": "https://example.com/test-addon.zip",
  "dependencies": [
    "https://example.com/library-one.json"
  ],
  "preserve": ["lua/autorun/myaddon_config.lua", "data", "materials/custom/**"]
}
```

Поля:

- `info.name` - название аддона
- `info.description` - короткое описание
- `info.author` - автор
- `version` - версия установленного и удалённого аддона
- `url` - прямая ссылка на архив с аддоном
- `dependencies` - список URL метаданных зависимостей
- `preserve` - файлы и пути, которые нельзя перезаписывать при обновлении

`preserve` поддерживает:

- отдельный файл, например `lua/autorun/myaddon_config.lua`
- целую папку, например `data`
- glob-выражение, например `materials/custom/**`

Все пути относительны к папке аддона.

Приложение ожидает только `http` или `https` URL. `README.md` всегда берётся локально из папки аддона, если файл существует.

## 🗂️ Источники и примеры

Источники хранятся списком URL индексов источников.

Пример:

```json
[
  "https://example.com/default_source.json"
]
```

Если нужен живой пример, смотри папку `exampleMeta/` в этом репозитории. Там лежат примерные метаданные для `mantle`, `newspawnmenu` и `thirdperson`. Встроенный `default_source.json` показывает пример индекса, который указывает на эти файлы.

Индекс источника и сами метаданные можно держать где угодно, если по `http` или `https` отдаётся валидный JSON.

## ⚙️ Установка и обновления

Логика одна и та же: приложение всегда опирается на метаданные аддона.

- install: сначала собирается план зависимостей, потом скачивается архив из `url`
- update: метаданные повторно загружаются по сохранённому URL и сравнивается `version`
- match: аддон уже актуален
- mismatch: аддон можно обновить

После установки GLuaManager сохраняет URL метаданных рядом с аддоном, чтобы проверка обновлений оставалась привязана к той же записи.

## 🛠️ Требования для сборки

На Debian / Ubuntu Linux нужны такие пакеты:

```bash
sudo apt update
sudo apt install libwebkit2gtk-4.1-dev \
  build-essential \
  curl \
  wget \
  file \
  libxdo-dev \
  libssl-dev \
  libayatana-appindicator3-dev \
  librsvg2-dev
```

Ещё нужен `tauri-cli`:

```bash
cargo install tauri-cli
```

Установите зависимости фронтенда:

```bash
cd ui
npm install
```

## ▶️ Запуск

Разработка:

```bash
cd src-tauri
cargo tauri dev
```

Сборка:

```bash
cd src-tauri
cargo tauri build
```
