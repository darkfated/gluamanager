# GLuaManager

Десктопный менеджер аддонов для Garry's Mod.

Язык: [🇷🇺 Русский](./README_RU.md) | [🇺🇸 English](./README.md)

GLuaManager помогает держать папку с аддонами в порядке. Он может показать, что уже установлено, что доступно для установки, и что пора обновить.

## 🔥 Как это работает

Есть два пути добавить аддоны в GLuaManager: сохранённые индексы источников и прямая ссылка на метаданные.

### 🔗 Поток данных

Схема простая:

- **source** это JSON-список ссылок на метаданные аддонов
- в каждом файле метаданных есть информация об аддоне и прямая ссылка на архив
- архив скачивается и распаковывается

Если известна одна ссылка на аддон, её можно установить напрямую.

### 📦 Установленные аддоны

- выберите папку `addons` Garry's Mod
- приложение смотрит только на непосредственные подпапки и только если в них есть `.addon`
- у каждого установленного аддона есть собственные локальные метаданные
- рядом сохраняется ссылка на метаданные
- при проверке обновлений приложение загружает эту ссылку и сравнивает версии
- если версии отличаются, аддон помечается как готовый к обновлению
- при обновлении приложение скачивает архив из поля `url` и накатывает его поверх локальной папки

### 🌐 Удалённый каталог

- добавьте один или несколько индексов источников в настройках
- каждый источник указывает на ссылки метаданных аддонов
- приложение загружает эти файлы и собирает каталог доступных аддонов
- в карточке показываются детали, ссылка на архив, зависимости и локальный README, если он есть
- установка сначала показывает дерево зависимостей, а потом скачивает только выбранное

> [!NOTE]
> GLuaManager не является package manager. Аддоны Garry's Mod живут в одном файловом пространстве, поэтому приложение делает упор на видимость, предупреждения о зависимостях и управляемую установку.

## 🧩 Формат метаданных

`.addon` это обычный JSON-файл в корне аддона.

Пример:

```json
{
  "info": {
    "name": "Test addon",
    "description": "Пример аддона для документации",
    "author": "darkfated"
  },
  "version": "1.0.0",
  "url": "https://example.com/test-addon.zip",
  "dependencies": [
    "https://example.com/library-one.json"
  ],
  "preserve": ["lua/autorun/myaddon_config.lua", "data", "materials/custom/**"]
}
```

Поля:

- `info.name` - название аддона
- `info.description` - короткое описание
- `info.author` - автор
- `version` - версия установленного и удалённого аддона
- `url` - прямая ссылка на архив с аддоном
- `dependencies` - список URL метаданных зависимостей
- `preserve` - файлы и пути, которые нельзя перезаписывать при обновлении

`preserve` поддерживает:

- отдельный файл, например `lua/autorun/myaddon_config.lua`
- целую папку, например `data`
- glob-выражение, например `materials/custom/**`

Все пути относительны к папке аддона.

Приложение ожидает только `http` или `https` URL. `README.md` всегда берётся локально из папки аддона, если файл существует.

## 🗂️ Источники и примеры

Источники хранятся списком URL индексов источников.

Пример:

```json
[
  "https://example.com/default_source.json"
]
```

Если нужен живой пример, смотри папку `exampleMeta/` в этом репозитории. Там лежат примерные метаданные для `mantle`, `newspawnmenu` и `thirdperson`. Встроенный `default_source.json` показывает пример индекса, который указывает на эти файлы.

Индекс источника и сами метаданные можно держать где угодно, если по `http` или `https` отдаётся валидный JSON.

## ⚙️ Установка и обновления

Всё начинается с файла метаданных аддона.

Ставите аддон - сначала увидите дерево зависимостей. Можно сразу оставить только нужное, и тогда скачивается именно выбранное.
Обновляете аддон - сначала идёт проверка текущей версии, а потом, если есть новее, программа предлагает её поставить.

После установки ссылка на метаданные остаётся рядом с аддоном, поэтому будущие проверки всегда смотрят на ту же запись.

## 💻 CLI

GLuaManager можно открыть и из терминала. Для этого используется та же команда `gluamanager`.

- если вы уже находитесь в папке `addons`, она считается рабочей автоматически
- `scan` показывает аддоны, которые уже лежат в этой папке
- `available` показывает аддоны из сохранённых источников, включая короткий `id`
- `install` принимает либо прямую ссылку, либо `addonId`
- `show` показывает красивую карточку с данными аддона
- `update` проверяет аддон и предлагает поставить свежую версию
- `rollback` возвращает последний сохранённый вариант
- `remove` удаляет аддон с подтверждением
- Linux-пакеты добавляют команду `gluamanager` в `/usr/local/bin`
- Windows-установщик добавляет папку программы в `PATH`

Типичные команды:

```bash
gluamanager scan
gluamanager available
gluamanager install test-addon
gluamanager install https://example.com/test-addon.json
gluamanager show my-addon
gluamanager update my-addon
gluamanager rollback my-addon
gluamanager remove my-addon
gluamanager --json scan
```

Ставьте `--json` перед командой, если нужен машинный вывод.

Полный список команд и примеров можно посмотреть через `gluamanager --help` или `gluamanager <команда> --help`.

## 🛠️ Требования для сборки

На Debian / Ubuntu Linux нужны такие пакеты:

```bash
sudo apt update
sudo apt install libwebkit2gtk-4.1-dev \
  build-essential \
  curl \
  wget \
  file \
  libxdo-dev \
  libssl-dev \
  libayatana-appindicator3-dev \
  librsvg2-dev
```

Ещё нужен `tauri-cli`:

```bash
cargo install tauri-cli
```

Установите зависимости фронтенда:

```bash
cd ui
npm install
```

## ▶️ Запуск

Разработка:

```bash
cd src-tauri
cargo tauri dev
```

Сборка:

```bash
cd src-tauri
cargo tauri build
```
